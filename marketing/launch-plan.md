# リリース初月 集客プラン — QuestBoard

**期間**: リリース日 〜 Day 30
**ターゲット**: 30代男性・「夢と日常をつなげたい」層
**KGI（最終目標）**: 初月1,000インストール・レビュー20件以上

---

## フェーズ概要

| フェーズ | 期間 | テーマ | 主な施策 |
|---|---|---|---|
| **0: 仕込み** | リリース2週前〜前日 | 期待値を高める | ティザー投稿・リスト整備 |

| **2: 拡散** | Day 4〜14 | バイラルを回す | シェアコード活用・SNS継続 |
| **3: 定着** | Day 15〜30 | レビュー獲得・継続利用 | プッシュ通知・コミュニティ |

## Phase 0: 仕込み（リリース2週前〜前日）

### SNS 事前準備

- [ ] X（Twitter）アカウントを開設し、プロフィールを整備
  - Bio例: `「いつか」を「今日」に変えるアプリ QuestBoard 開発中 | やりたいことリスト × TODO管理`
- [ ] ティザー投稿 × 3本（週2回ペース）
  - 投稿例1: 「30代になってから、メモ帳の夢リストを見返さなくなった。なぜかというと——（続く）」
  - 投稿例2: 「友達の"やりたいことリスト"を覗いたら、自分も気づいてなかったやりたいことが出てきた。そういうアプリを作っています。」
  - 投稿例3: 「もうすぐリリースします。バケットリスト × TODO管理。先行で触ってくれる人を5名募集。」
- [ ] 先行テスター5〜10名をDMで個別招待（TestFlight / Android内部テスト）

### ASO・ストア準備

- [ ] `aso.md` の説明文・キーワードをストアに設定
- [ ] スクリーンショット5枚を用意（1枚目に最重要ベネフィットを入れる）
- [ ] アプリプレビュー動画（15〜30秒）を用意できれば理想

## Phase 1: 爆発（Day 1〜3）

### Day 1：一斉告知

**X（Twitter）**
- リリース告知ツイートを投稿（固定ピン設定）
  ```
  【リリースしました】

  「いつかやろう」を「今日やること」に変えるアプリ
  QuestBoard、ついにリリース。

  バケットリストと日常TODOを一つのアプリで。
  シェアコードで友達の夢リストをインポートできます。

  無料なので、まず触ってみてください。
  → [ストアリンク]

  #やりたいことリスト #QuestBoard
  ```
- インフルエンサー・知人への個別DM（10〜20名）
  - 「触ってみてもらえませんか？正直なフィードバックが欲しいです」

**口コミ火付け（クローズドコミュニティ）**
- Slack / Discord（自己啓発・副業系コミュニティ）に投稿
- Facebook グループ（30代・ライフデザイン系）に投稿
- 知人グループLINEに送付

### Day 2〜3：フォロワー巻き込み

- リツイート・返信にお礼コメントで露出を上げる
- 「シェアコード交換しましょう」投稿でUGCを誘発
  ```
  QuestBoardのシェアコードを交換しましょう。
  自分のやりたいことリストをコードで貼ってください。
  見に行きます＆インポートします。
  ```

## Phase 2: 拡散（Day 4〜14）

### SNS 継続投稿（週3本ペース）

| 曜日 | 内容 | 目的 |
|---|---|---|
| 月曜朝 | 「今週やること」テーマで共感投稿 | 週初め層への訴求 |
| 水曜夜 | 機能紹介・使い方TIPS | 認知→インストール転換 |
| 日曜夜 | 「いつかやろう」系の哲学投稿 | 内省層への刺さり・ブックマーク獲得 |

### シェアコード活用施策

- 「シェアコードチャレンジ」を企画
  ```
  あなたの「今年やること10選」をシェアコードで投稿してみてください。
  #QuestBoardチャレンジ でシェア → 私が全員のリストをインポートしてRTします。
  ```
- UGC（ユーザー投稿）を積極的にRT・引用RT

### SEO（Web流入）

- **note 記事を2本書く**
  - 記事1:「30代がやりたいことリストを作るべき理由と、続けるためのコツ」
    - ターゲットキーワード: やりたいことリスト 作り方 / バケットリスト 30代
  - 記事2:「バケットリストと日常TODOを一緒に管理したら、仕事のモチベーションが変わった話」
    - ターゲットキーワード: バケットリスト アプリ / 夢管理 アプリ
  - 記事末尾に自然な形でQuestBoardを紹介・リンク挿入

- **Qiita / Zenn は不要**（エンジニア向けのため、ターゲット層と乖離）

## Phase 3: 定着（Day 15〜30）

### レビュー獲得

- アプリ内でのレビュー依頼ダイアログ（初回クエスト達成時 or 3日目起動時に表示）
- SNSで「レビューお願い投稿」
  ```
  QuestBoardをリリースして2週間。
  少しずつ使ってくれる人が増えてきました。

  正直な感想をApp Store / Google Play のレビューに書いてもらえると、
  開発の励みになります。良くても悪くても嬉しいです。
  ```

### リテンション施策

- **プッシュ通知**（週1回・Sunday 20時）
  - 「今週のクエスト、進んでますか？」
- **メールマガジン or X スペース**（月1回）
  - 「今月達成したこと発表会」をTwitterスペースで開催 → コミュニティ化の第一歩

### 有料転換（買い切り / サブスク）

- Day 14 頃に「プレミアム機能紹介」投稿
- 初月限定の早期割引（例: 買い切り¥3,800→初月限定）を検討
